1. `.claude/context/POKER_THEORY.md` — poker theory foundation (shared with `assumptionEngine/`, `exploitEngine/`, `rangeEngine/`)

6. `src/utils/assumptionEngine/CLAUDE.md` — parent engine's domain rules (apply by inheritance)

This module produces and maintains `ExploitAnchor` objects — named multi-street spot archetypes extending `VillainAssumption` v1.1 with perception-model primitives + GTO baseline decomposition. Consumers: three new study-mode surfaces (`AnchorLibraryView`, `CalibrationDashboardView`, `SessionReviewView`) + `HandReplayView` Section G capture + minimal `LiveAdviceBar` anchor badge.

The module is **structurally separate** from `assumptionEngine/` but inherits its schema + validator + simulator primitives. Anchors additively extend assumptions; they don't replace them.

## Core Principles

### 1. Anchors are VillainAssumptions + structural extensions (inheritance contract)

Every `ExploitAnchor` is a `VillainAssumption` (schema v1.1) plus the additive fields specified in `schema-delta.md` §2. Every v1.1 field retains its full specification and gating semantics. See schema-delta §1 for the inheritance contract.

**Rule:** anchor validation MUST pass every v1.1 gate that applies, PLUS the anchor-specific invariants I-EAL-1..9 from `schema-delta.md` §4.

### 2. Composite semver — base + extension bump independently

`schemaVersion` is a compound semver: `<base-version>-anchor-v<extension-version>` (e.g. `"1.1-anchor-v1.0"`). The base half tracks `VillainAssumption`'s version; the extension half tracks EAL-specific additions. Both bump independently per `gate4-p3-decisions.md` §1.

### 3. Three-tier model — capture / author / retire (Phase 1 MVP)

- **Tier 0** — owner one-click observation capture via `hand-replay-observation-capture` surface (W-AO-1 writer).
- **Tier 2** — AI-authored anchors from seed library; owner override/retire via `anchor-library` + `calibration-dashboard` + `anchor-retirement` journey.
- **Tier 3** — auto-retirement at session-close when `retirementCondition` fires (observed-rate credible interval overlaps GTO baseline).
- **Tier 1** (candidate promotion from clustered observations) is Phase 2 scope; W-AC-1/2/3 writers are placeholder stubs.
- **From-scratch authoring UI** — out of scope indefinitely.

### 4. Perception primitives have cross-anchor invalidation atomicity (I-EAL-9, I-WR-3)

When a `PerceptionPrimitive.validityScore` posterior credible interval crosses 0.5, **every** `ExploitAnchor` that references the primitive is flagged **atomically** via a single IDB transaction. Partial commit is forbidden. `dependentAnchorCount` is an enforced invariant. See `schema-delta.md` §3.3.1.

### 5. Live-surface footprint is headless — classifier-fired badge only

`LiveAdviceBar` anchor badge shows `archetypeName + confidence dial` only. No calibration state, no retirement progress, no trend arrows. Cross-surface contamination is refused (anti-pattern AP-07 + red line #8). Calibration deep-dive lives on `calibration-dashboard` study surface only.

### 6. Copy-discipline is CI-linted (AP-06 refusal)

Calibration dashboard's model-accuracy prose + retirement journey's confirm sheet copy are generated by **deterministic** `calibrationCopy.js` + `retirementCopy.js` modules. Runtime LLM generation is refused. CI grep check on forbidden strings ("your accuracy" / "your observation accuracy" / "score" / "grade" / "you were off") prevents drift. See `docs/projects/exploit-anchor-library/anti-patterns.md` §AP-06.

### 7. Signal separation — matcher-observations and owner-captures never fuse (I-AE-7 inherited, AP-08)

`AnchorObservation` records carry `origin: "matcher-system" | "owner-captured"`. Tier 2 calibration math can weight them differently, but they are **never** arithmetically summed into a single "evidence count." Enforced at writer boundary (I-WR-2) AND at render level on `calibration-dashboard` + `session-review-anchor-rollup` surfaces.

### 8. Incognito observation mode is structurally non-negotiable (Red Line #9)

Per-observation `contributesToCalibration: boolean` flag is always respected. Opt-out default when `observation_enrollment_state === 'enrolled'` (per Q2-A). Capture modal's Incognito toggle is **always primary-visible**, forced-on-disabled when `not-enrolled`. See `gate3-owner-interview.md` §Q2 + capture surface spec.

### 9. Enumerable writer registry — CI-grep enforced (I-WR-1)

Every write to `exploitAnchors` / `anchorObservations` / `anchorCandidates` / `perceptionPrimitives` MUST be listed in `docs/projects/exploit-anchor-library/WRITERS.md`. CI script `scripts/check-anchor-writers.sh` (Phase 5) fails on unregistered writes. Mirrors `scripts/check-sidebar-writers.sh` pattern from SR-6.

## Anti-Patterns

### DO NOT render a sub-threshold anchor (inherited from I-AE-1)

Four-gate quality (posteriorConfidence ≥ 0.80, stability composite ≥ 0.70, surface-specific recognizability, asymmetric payoff ≥ 0.30) PLUS Sharpe floor ≥ 1.0 apply to anchors just as to assumptions. Sub-threshold anchors stay as `candidate` status and do NOT fire on any surface.

### DO NOT import from `exploitEngine/` outside explicit sanctioned paths

Inherits I-AE-CIRC-1 from parent engine. Anchors are behavioral archetypes; they don't reach into exploit-logic. Only `gameTreeEvaluator` is a sanctioned dependency from `exploitEngine/`.

### DO NOT bypass signal separation (AP-08)

Never sum matcher-observed and owner-captured observation counts. Never compute a blended "evidence score." Tier 2 calibration math applies per-origin weights that are posted parameters in the transparency screen.

### DO NOT surface retirement reconsideration nudges (AP-05)

Retired anchors remain visible in the flat-list per red line #6, but the system NEVER surfaces "retired anchors you might reconsider" affordances. Owner un-retirement requires explicit action on `anchor-library` with 2-tap confirmation.

### DO NOT render calibration state on live surfaces (AP-07, Red Line #8)

`LiveAdviceBar` anchor badge shows `archetypeName + confidence dial` only. Observed rate / predicted rate / credible interval / retirement state / trend arrow are ALL study-mode-only. Cross-surface contamination is a structural failure.

### DO NOT fabricate a GTO baseline

Every anchor's `gtoBaseline` must reference a valid `method` from schema-delta §2.4 enum (`MDF` / `pot-odds-equilibrium` / `hand-down-equity` / `range-balance`). The `referenceRate` + `referenceEv` must be derivable from the method + game-state. If the math doesn't check out in the Tier 1 scenario, the anchor doesn't ship.

### DO NOT grade the observer (AP-06)

Copy that evaluates the owner's observations — "your accuracy," "your observations didn't match," "you misjudged this spot" — is categorically forbidden. Calibration Dashboard evaluates the MODEL's prediction; it never grades Chris. Copy-discipline generators enforce at CI time.

### DO NOT auto-retire mid-session

Tier 3 retirement evaluator runs at **session-close only**. No mid-session surprise state transitions (Nielsen H-N05 error prevention). User-initiated retirement via override action does take effect immediately, but system-initiated auto-retirement always waits for session boundary.

### DO NOT skip the signed-session-close boundary

Retirement evaluator reads all active anchors + computes posterior-CI-vs-GTO overlap + transitions status. It uses session-close as a natural transaction boundary. Never run the evaluator in a render loop, effect, or timer.

### DO NOT combine perception-primitive validity with anchor evidence arithmetically

Primitive validity (0..1) and anchor evidence (Beta posterior) are distinct inference objects on different scales. Combining them violates I-AE-7 signal separation. When a primitive's validity degrades, dependent anchors are flagged + `quality.composite` penalty factor applies — but not by arithmetic fusion.

## Key Concepts

### Compound schema version
`<base>-anchor-v<ext>`. Base tracks VillainAssumption; extension tracks EAL additions. Independent bumps.

### GTO baseline (anchor-specific)
One of `MDF` / `pot-odds-equilibrium` / `hand-down-equity` / `range-balance`. `referenceRate` + `referenceEv` are derived from method + game-state. Tier 1 scenario validates math.

### EV decomposition
`statAttributable + perceptionAttributable = 1.0` (floating-point tolerance ≤ 0.01). Assigns share of observed edge to raw statistical deviation vs perception gap. Validator enforces sum.

### Retirement condition
`credible-interval-overlap` is the Phase 1 default method. Retires anchor when observed-rate CI crosses GTO `referenceRate`. Evaluated at session-close.

### Primitive validity posterior
Bayesian posterior on the primitive's load-bearing claim. Tier 2 metric updated on each dependent anchor's firing (W-PP-2 writer). Cross-anchor invalidation fires when CI crosses 0.5.

### Perception-primitive dependent count invariant (I-EAL-9)
`PerceptionPrimitive.validityScore.dependentAnchorCount` MUST equal `count(anchors where perceptionPrimitiveIds includes this.id)`. Recomputed at every anchor write; referential consistency checked at every primitive write.

### Tier 1 math-integrity scenario (anchor-level)
Extends `assumptionEngine/__sim__/scenarioRunner.js` pattern: synthetic villain with exactly-coded policy + injected producer + expected vs simulated dividend within 5% + GTO baseline math verification + honesty check.

### Newcomer unlock threshold
`CONSTANTS.ANCHOR_LIBRARY_UNLOCK_THRESHOLD = 25` hands. Editable in `src/constants/anchorLibraryConstants.js` per `gate4-p3-decisions.md` §5. Hard cutoff; not gradual unlock.
