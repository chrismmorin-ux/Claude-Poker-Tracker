## How to review

For each evaluator surface, verify:

1. **The surface description is accurate** — does the cited file/function actually grade LLM output?
2. **The ground-truth anchor is honest** — is the evaluator anchored to peer-reviewed reference (strong), founder-approved examples (weak), or LLM consensus (weakest, recursively self-justifying)?
3. **The spot-check cadence exists** — is there an actual scheduled human-expert review, or is this aspirational?
4. **The divergence metric is defined** — what number tells you the evaluator has drifted?
5. **The calibration history is recorded** — when was the evaluator last re-calibrated against fresh expert review?
6. **Failure modes are surfaced** — what happens if the evaluator approves a wrong claim?

If the answer to any of (2)-(6) is "engine could not determine" or "to be defined", that section needs founder input before promotion.

> **Strength rubric:**
> - **Strong:** anchored to authoritative external reference (peer-reviewed, FDA monograph, settled case law, validated SymPy proof)
> - **Medium:** anchored to authoritative reference but with a known degradation path (e.g., textbook that may be stale)
> - **Weak:** anchored to founder-approved examples (encodes founder bias over time)
> - **Weakest:** anchored to LLM consensus or other self-referential pattern
