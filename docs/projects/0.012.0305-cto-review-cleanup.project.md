### Goal

Eliminate double Bayesian computation. Currently `usePlayerTendencies` and `useRangeProfile` independently call `buildRangeProfile()` for the same player.

**Depends on:** Phase 2 complete (both touch PlayerContext surface area)

### Tasks

#### T-3A: Map current computation flow (~10 min)

Read and document how each hook builds profiles:
- `src/hooks/usePlayerTendencies.js` — builds `tendencyMap` with `rangeProfile` + `rangeSummary` per player
- `src/hooks/useRangeProfile.js` — independently calls `buildRangeProfile()` + `saveRangeProfile()`

Identify: Where does `useRangeProfile` get consumed? (AnalysisView, StatsView?)

#### T-3B: Implement cache-first strategy (~30 min)

**Approach (choose during T-3A):**

- **Option A (preferred):** Expose cached range profiles from `usePlayerTendencies` via PlayerContext. `useRangeProfile` checks context cache before computing.
- **Option B:** `useRangeProfile` accepts an optional pre-built profile param, skipping `buildRangeProfile()` when provided.

**Files:**
- `src/hooks/useRangeProfile.js` — add cache check before `buildRangeProfile()`
- `src/hooks/usePlayerTendencies.js` — may need to expose profile cache
- `src/contexts/PlayerContext.jsx` — may need to pass profile cache through
