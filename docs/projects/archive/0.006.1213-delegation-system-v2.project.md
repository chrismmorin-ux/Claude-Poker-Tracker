## Overview

**Goal:** Transform the local model delegation system from "try to preserve files" to "tests define success."

**Key Insight:** If local models replace files entirely, that's fine IF tests pass. This eliminates Type D task failures by reframing the problem.

- TDD-first paradigm shift (foundational)
- Modular policy documentation (maintainable)
- Quick iteration system (tunable prompts/limits)

## Phase 1: TDD-First Foundation [x] COMPLETE

### Goal

Establish the paradigm shift: **Tests define success, not file preservation.**

### The Insight

**Old thinking:** Type D tasks → Escalate (local models fail at "modify")
**New thinking:** Type D tasks → Decompose to "write new with test"

```
Task structure:
1. Write failing test first (defines expected behavior)
2. Local model implements (can replace entire file - we don't care)
3. Test passes → Accept
4. Test fails → Iterate or escalate
```
