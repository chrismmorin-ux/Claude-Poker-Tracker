## Problem

`postflopNarrower.js` lines 244-249 uses hardcoded per-bucket action multipliers:

```js
const ACTION_MULTIPLIERS = {
  raise:  { nuts: 1.0, strong: 0.8, marginal: 0.1, draw: 0.4, air: 0.15 },
  call:   { nuts: 0.2, strong: 0.8, marginal: 0.9, draw: 0.8, air: 0.1 },
  check:  { nuts: 0.3, strong: 0.4, marginal: 0.7, draw: 0.5, air: 0.8 },
  bet:    { nuts: 0.9, strong: 0.9, marginal: 0.3, draw: 0.4, air: 0.2 },
};
```

**Two problems:**

1. **No within-bucket granularity.** A 15-out combo draw and an 8-out gutshot+overcards both get `0.4` for raise. A gutshot (4 outs, classified as "air" since <8) gets `0.15` but should semi-bluff more than pure nothing. An overpair and TPTK both get `0.8` for raise but the overpair raises more.

2. **No player adaptation.** A calling station and a TAG get identical multipliers. A player with AF 0.3 should have higher call multipliers; AF 3.0 should have higher raise multipliers.

## Approach

Two layers of improvement, both backward-compatible:

**Layer 1 — Continuous multipliers:** Replace bucket lookup with a function that uses `category`, `score`, and `drawOuts` to compute a continuous multiplier. Uses existing bucket multipliers as anchor points but interpolates within buckets based on draw outs and hand strength.

**Layer 2 — Player adaptation:** Optional `playerStats` parameter that adjusts multipliers based on observed tendencies (AF, VPIP, fold frequency).

## Implementation

### Step 1: Refactor `classifyCombo` to avoid double computation

Currently `narrowByBoard` calls `classifyCombo` which internally calls `evaluateComboStrength` and `detectDraws`. But for continuous multipliers we need access to `score`, `category`, and `drawOuts` — not just the bucket name.

**Create `classifyComboFull(card1, card2, board)`** that returns:
```js
{ bucket: string, score: number, category: number, drawOuts: number }
```

Refactor existing `classifyCombo` to be a thin wrapper:
```js
export const classifyCombo = (c1, c2, board) => classifyComboFull(c1, c2, board).bucket;
```

No external behavior change. All existing tests pass unchanged.

### Step 2: Create `comboMultiplier(action, { category, score, drawOuts })`

Continuous multiplier function. Uses `ACTION_MULTIPLIERS` as anchor points with draw-outs-based interpolation.

```js
// Per-action scaling: how much each draw out adjusts the multiplier
const OUTS_SCALING = {
  raise: 0.025,   // 8 outs: +0.20, 15 outs: +0.375 (semi-bluff)
  call:  0.015,   // 8 outs: +0.12, 15 outs: +0.225 (pot odds)
  check: -0.02,   // more outs -> less likely to just check
  bet:   0.025,   // semi-bluff sizing
};
```

**Logic:**
1. Look up base multiplier from `ACTION_MULTIPLIERS[action][bucket]`
2. If combo has `drawOuts > 0` and bucket is `'draw'` or `'air'`:
   - Apply `outsBonus = drawOuts * OUTS_SCALING[action]`
   - Return `clamp(base + outsBonus, 0, 1)`
3. Otherwise return the base multiplier (score-within-category refinement is a future enhancement)

**Key improvement:** Air hands with 4-7 outs (gutshots, overcards) now get meaningful bumps instead of flat air multiplier. A gutshot (4 outs) classified as air gets `0.15 + 4*0.025 = 0.25` for raise instead of flat `0.15`.

### Step 3: Create `adaptMultipliers(playerStats)`

Generates player-specific multiplier adjustments.

**Params:**
```js
{
  af: number,            // aggression factor (optional)
  foldToCbet: number,    // fold to c-bet % 0-100 (optional)
  vpip: number,          // VPIP % 0-100 (optional)
}
```

**Returns:** Object with same shape as `ACTION_MULTIPLIERS`, values adjusted from defaults.

**Adjustment rules:**
- `af > 2.0`: raise multipliers for strong/draw +0.1, call multipliers -0.1
- `af < 0.8`: call multipliers +0.15, raise multipliers -0.15
- `foldToCbet > 60`: check/fold multipliers for air/draw +0.1 (gives up more)
- `vpip < 20`: call multipliers for marginal/draw -0.1 (tighter continuing range)
- All values clamped to `[0.05, 1.0]`

### Step 4: Update `narrowByBoard` inner loop

Replace:
```js
const bucket = classifyCombo(c1, c2, board);
totalMultiplier += multipliers[bucket];
```

With:
```js
const { bucket, score, category, drawOuts } = classifyComboFull(c1, c2, board);
totalMultiplier += comboMultiplier(action, { category, score, drawOuts });
```
