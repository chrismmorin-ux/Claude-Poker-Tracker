**Status:** harness shipped (SPR-053); first run **pending — founder runs in browser**.

## How to produce this baseline

1. `npm run dev`
2. Open http://localhost:5173 in a browser (any 9-max poker tracker view loads)
3. Open the dev console
4. Run BTN first (~10–15 min):
   ```js
   const btn = await window.__validateMWEquity('BTN')
   console.log(btn.report)  // markdown — copy into this file under "## BTN baseline"
   ```
5. Inspect the divergence numbers. Sanity-check:
   - AA must be in derived range. ✓
   - KK must be in derived range. ✓
   - 72o EV inspected (BTN reference includes loose hands).
6. Then run UTG (~10 min):
   ```js
   const utg = await window.__validateMWEquity('UTG')
   console.log(utg.report)  // copy into "## UTG baseline"
   ```
7. Founder review checkpoint:
   - **Jaccard overlap > 70%** → "close enough" → normal /next queue. Document anomalies in v2 follow-up tickets.
   - **Jaccard overlap 40–70%** → "moderate divergence" → triage; promote highest-confound-source improvements to P1 or P2.
   - **Jaccard overlap < 40%** → "significantly off" → P0/P1 fix required. Most likely engine math or joint-probability table error. Read the confound section to identify which.
