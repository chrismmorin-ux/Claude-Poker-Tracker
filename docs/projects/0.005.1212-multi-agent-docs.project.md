## Overview

We implemented a major architectural change (Multi-Agent Compliance Architecture) but:
1. **Documentation not updated** - All docs still describe old direct-execution workflow
2. **Hooks conflict** - 8+ delegation hooks now conflict with permission-based enforcement
3. **Friction points** - write-gate.cjs blocks plan files, multiple redundant checks

### The Change That Was Made

**OLD ARCHITECTURE:**
```
Claude → Decomposes → Creates LOCAL_TASKS → Executes scripts directly
         ↓
    Hooks enforce (delegation-check-pre.cjs, write-gate.cjs, etc.)
```

**NEW ARCHITECTURE:**
```
Claude Primary (NO WRITE PERMISSIONS)
         ↓
    Task(dispatcher) → Dispatcher Agent evaluates
         ↓                    ↓
    Local model OK     Escalation needed
         ↓                    ↓
    Dispatcher executes  Human approval → Worker Agent executes
```

## Phase 1: Resolve Hook Conflicts ← CURRENT

### Goal
Fix immediate friction: plan files blocked, redundant enforcement creating confusion.

### Problem Analysis

**write-gate.cjs Issues:**
1. Blocks ALL writes unless backlog task exists
2. No exception for `.claude/plans/**` (plan mode files)
3. No exception for `docs/projects/*.project.md` (project files)
4. Conflicts with settings.local.json which ALLOWS these paths

**Redundancy:**
- Permissions block: `Write(src/**)`, `Edit(src/**)`, `Write(scripts/**)`, etc.
- write-gate.cjs: Also blocks these same paths
- delegation-check-pre.cjs: Also blocks these same paths
- Result: Triple-checking, but permissions are authoritative

### Resolution: Consolidate Hooks

Keep write-gate.cjs but:
1. Add exceptions for plan/project files
2. Remove redundant delegation hooks
3. Keep backlog-task requirement for source files only

### T-MAD-001 Implementation Details

Add to write-gate.cjs `checkBypassFile()` function or create new function:

```javascript
// Files that bypass backlog task requirement
const BYPASS_PATTERNS = [
  /^\.claude\/plans\//,           // Plan files
  /^docs\/projects\/.*\.project\.md$/,  // Project files
  /^\.claude\/dispatcher-state\.json$/,  // Dispatcher state
];

function isExemptFile(filePath) {
  const relative = getRelativePath(filePath);
  return BYPASS_PATTERNS.some(pattern => pattern.test(relative));
}
```

### Key Content to Add to WORKFLOW.md

```markdown
## Multi-Agent Architecture

Claude Primary (this session) has NO file write permissions.
All modifications must route through the Dispatcher agent.

### Agent Roles

| Agent | Model | Purpose | Permissions |
|-------|-------|---------|-------------|
| Primary | opus | Plan, decompose, request | Read, Search, TodoWrite |
| Dispatcher | haiku | Evaluate, execute, escalate | Read, Write, Edit, Bash |
| Worker | sonnet | Execute approved escalations | Task-specific files only |

### Workflow

1. Claude Primary decomposes task into atomic pieces
2. Claude Primary calls `Task(subagent_type="dispatcher")`
3. Dispatcher evaluates: Is task in backlog? Assigned? Policy-compliant?
4. Dispatcher executes via local model OR creates permission request
5. If human approves, Dispatcher spawns Worker agent
6. Worker executes single task and terminates

### Permission Structure (settings.local.json)

**Claude Primary CAN:**
- Read any file
- Search (Glob, Grep)
- Web search/fetch
- Create todos
- Call Dispatcher agent
- Write to `.claude/plans/**` and `docs/projects/*.project.md`

**Claude Primary CANNOT:**
- Write/Edit source files (`src/**`, `scripts/**`)
- Write/Edit hooks (`.claude/hooks/**`)
- Write/Edit config files (`.claude/settings*.json`, etc.)
- Use file-modifying Bash commands
- Spawn Worker agent directly

### Human Approval CLI

```bash
./scripts/approve.sh              # View pending requests
./scripts/approve.sh approve ID   # Approve request
./scripts/approve.sh deny ID      # Deny request
./scripts/approve-menu.sh         # Interactive menu
```
```
