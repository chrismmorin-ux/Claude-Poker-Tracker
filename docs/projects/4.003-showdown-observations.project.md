# Project 4.003 — Showdown Observations for Bayesian Range Refinement

**Status:** READY
**Priority:** P1 (unblocks backlog item 4b)
**Scope:** 4 files modified, ~120 lines added, 1 test file added

---

## Problem

The range engine treats all showdown anchors identically — a player who showed AKs and **won** gets the same boost as one who showed AKs and **lost**. This loses valuable information:

- **Winning** at showdown confirms the hand was strong enough for the action + street sequence
- **Losing** at showdown suggests the hand may have been toward the bottom of that action's range
- **Mucking** means no cards were revealed — no data to extract

The infrastructure is 70% built. Cards are captured in `cardState.allPlayerCards[seat]`, persisted to IndexedDB, and `actionExtractor.js` already extracts `showdownIndex`. The gap is outcome-aware processing.

---

## Current Data Flow

```
ShowdownView (UI)
  -> user assigns cards + clicks Muck/Won
  -> dispatchGame(RECORD_SHOWDOWN_ACTION) → actionSequence entry: { seat, action: 'mucked'|'won', street: 'showdown' }
  -> dispatchCard(SET_PLAYER_CARD) → cardState.allPlayerCards[seat] = [card1, card2]
  -> nextHand() → saveHand() → IndexedDB

buildRangeProfile(playerId, hands)
  -> extractAllActions() → returns { position, rangeAction, facedRaise, showdownIndex }
  -> updateProfileFromActions() → counts + Bayesian grid updates
  -> applyShowdownAnchor() → sets confirmed hand to 1.0, boosts neighbors
  -> normalizeAllPositions()
```

**Gap:** `extractPreflopAction()` extracts `showdownIndex` (the hand) but ignores whether the player won or lost. `applyShowdownAnchor()` applies identical boosts regardless of outcome.

## Plan

### Phase 1: Extract showdown outcome (actionExtractor.js)

**File:** `src/utils/rangeEngine/actionExtractor.js`

Add outcome extraction to `extractPreflopAction()`:

```js
// After extracting showdownIndex (line 64-77), also extract outcome
let showdownOutcome = null; // 'won' | 'lost' | null
if (showdownIndex !== null) {
  const showdownActions = (hand.gameState?.actionSequence || [])
    .filter(e => e.street === 'showdown' && e.seat === seat);
  const wonAction = showdownActions.find(e => e.action === ACTIONS.WON);
  showdownOutcome = wonAction ? 'won' : 'lost';
}

return { position, rangeAction, facedRaise, showdownIndex, showdownOutcome };
```

**Why here:** Keeps all per-hand extraction in one place. No new module needed.

**Import needed:** `ACTIONS` from `../../constants/gameConstants`

### Phase 2: Outcome-aware anchoring (bayesianUpdater.js)

**File:** `src/utils/rangeEngine/bayesianUpdater.js`

Modify `applyShowdownAnchor()` to accept and use outcome:

```js
applyShowdownAnchor(profile, position, action, handGridIndex, outcome = null)
```

**Boost strategy by outcome:**

| Outcome | Confirmed hand | Alt suitedness | Adjacent pairs | Adjacent kickers |
|---------|---------------|----------------|----------------|------------------|
| Won     | 1.0           | +0.30          | +0.25          | +0.20            |
| Lost    | 1.0           | +0.15          | +0.10          | +0.08            |
| null (legacy) | 1.0     | +0.25          | +0.20          | +0.15            |

**Rationale:**
- Confirmed hand is always 1.0 (we *saw* it)
- Won = player's range at this position/action skews strong → boost similar hands more
- Lost = hand was in range but toward the bottom → still boost (they play it) but less aggressively
- null = backward compatible with existing anchors that lack outcome data

Also store outcome in the anchor record:
```js
profile.showdownAnchors.push({
  position, action, handIndex: handGridIndex, outcome,
});
```

**Caller update in `updateProfileFromActions()`:**
```js
// Line 76: pass outcome through
applyShowdownAnchor(profile, ext.position, ext.rangeAction, ext.showdownIndex, ext.showdownOutcome);
```

## Edge Cases

- **Player mucks (no cards assigned):** `allPlayerCards[seat]` is empty → `showdownIndex = null` → no anchor applied. Correct.
- **Multiple winners:** Each WON seat gets its own anchor independently. No conflict.
- **Cards assigned but no showdown action recorded:** `showdownOutcome` defaults to `'lost'` (conservative). Cards are only assigned in ShowdownView which always records muck/won.
- **Legacy hands without outcome data:** `showdownOutcome` will be null → original boost values used. Fully backward compatible.
