# Project: Claude Infrastructure Cleanup

## Problem Statement

The `.claude/` infrastructure has grown to dwarf the actual application (~4.7K LOC React app). Currently:

- **25 hooks** fire on nearly every tool call (Read, Write, Edit, Grep, Glob, Bash, Task, SlashCommand)
- **24 slash commands** (most never used)
- **8 agent definitions**
- **15+ scripts** in `scripts/` (analytics, reporting, efficiency)
- **22 JSON state/tracking files** in `.claude/`
- **settings.json** is 315 lines of hook configuration

Every user prompt triggers 4 hooks (startup-menu, budget-check, pm-project-required, efficiency-advisor). Every file read triggers 4 hooks (scan-first-check, cache-first-read, efficiency-tracker, metrics-collector). Every edit triggers 7 hooks. The token budget tracker permanently shows "EXCEEDED" because the budget estimate system doesn't track real tokens — it uses fake estimates that accumulate faster than actual usage.

The infrastructure designed to improve efficiency is the primary source of inefficiency.

## Goal

Strip `.claude/` to only what demonstrably adds value. Target: a setup that a solo developer on a ~5K LOC project would actually benefit from.

## Guiding Principles

1. **If it doesn't prevent a real mistake or save real time, remove it**
2. **Hooks should be rare safety nets, not constant companions**
3. **Commands should be things you actually type, not theoretical capabilities**
4. **Context files are valuable only if they save more tokens than they cost to maintain**

## Inventory & Decisions

### Hooks (25 total → 3 kept)

#### KEEP — Safety hooks (fire only on specific dangerous commands)

| Hook | Trigger | Why Keep |
|------|---------|----------|
| `secrets-scan.cjs` | PreToolUse:Bash (git add/commit only) | Prevents committing .env, credentials, private keys |
| `git-guard.cjs` | PreToolUse:Bash (git push/reset only) | Prevents force-push to main, hard reset on main |
| `pre-commit-gate.cjs` | PreToolUse:Bash (git commit only) | Suggests review before large commits (non-blocking) |

These three hooks are lightweight, trigger only on specific git commands, and prevent real mistakes.

#### REMOVE — 22 hooks

| Hook | Trigger | Why Remove |
|------|---------|------------|
| `startup-menu.cjs` | UserPromptSubmit | ASCII menu on every conversation start. Wastes ~200 tokens per session. User can just ask for what they want. |
| `budget-check.cjs` | UserPromptSubmit | 384 lines. Uses fake token estimates, permanently shows "EXCEEDED". Adds noise to every prompt. |
| `pm-project-required.cjs` | UserPromptSubmit | Forces project file for any work. Blocks simple tasks unnecessarily. |
| `efficiency-advisor.cjs` | UserPromptSubmit | Injects efficiency tips. Meta-advice instead of doing work. |
| `efficiency-tracker.cjs` | PostToolUse:Edit,Write,Read,Task | Fires on every read/write. Tracks fake efficiency metrics. |
| `metrics-collector.cjs` | PostToolUse:Edit,Write,Read,Bash,Grep,Glob,Task | Fires on literally every tool call. Writes JSON. Pure overhead. |
| `scan-first-check.cjs` | PreToolUse:Read,Grep,Glob | Suggests reading index files first. Slows down normal searching. |
| `cache-first-read.cjs` | PreToolUse:Read | Caches file reads in JSON. Unnecessary — Claude Code handles context. |
| `index-first-check.cjs` | PreToolUse:Task | Forces index check before Task tool. Unnecessary gate. |
| `plan-mode-edit-check.cjs` | PreToolUse:Write,Edit | Blocks edits outside plan mode for multi-file changes. Annoying for small fixes. |
| `pm-time-estimate-detector.cjs` | PostToolUse:Edit,Write | Detects time estimates in written content. Irrelevant. |
| `docs-sync.cjs` | PostToolUse:Edit,Write | Tracks whether docs are stale after edits. Nice idea, but the overhead > value. |
| `pr-size-tracker.cjs` | PostToolUse:Edit | Tracks PR line count. No PRs in this workflow. |
| `agent-suggest.cjs` | PostToolUse:Edit,Write | Suggests using agents after edits. Meta-noise. |
| `project-update.cjs` | PostToolUse:Edit,Write | Updates project status JSON on edits. Overhead. |
| `new-file-test-reminder.cjs` | PostToolUse:Write | Reminds to write tests for new files. Can just ask. |
| `commit-project-check.cjs` | PreToolUse:Bash | Checks project alignment before commits. Overhead. |
| `test-tracker.cjs` | PostToolUse:Bash | Tracks test run history. Unnecessary. |
| `test-summary.cjs` | PostToolUse:Bash | Summarizes test output. smart-test-runner.sh already does this. |
| `lint-summary.cjs` | PostToolUse:Bash | Summarizes lint output. No linting configured. |
| `process-suggest.cjs` | PostToolUse:Bash | Suggests process improvements after commands. Meta-noise. |
| `audit-auto-capture.cjs` | PostToolUse:SlashCommand | Captures audits when slash commands run. No audit system needed. |
| `quality-gate.cjs` | PreToolUse:Bash | Quality checks on bash commands. Redundant with git-guard. |

- Removed all PostToolUse hooks (no more metrics, efficiency tracking, docs sync, agent suggestions)
